✅ SAPTHALA Boutique - Unified Database Integration

This template demonstrates how to properly integrate the database services with your Admin Panel, Sub-Admin Panel, and Staff Application. All services work together seamlessly with real-time synchronization and offline support.

✨ Key Features:
• Unified Database Service for all operations
• Secure Authentication Service with role-based access
• Real-time data synchronization across all panels
• Offline support with intelligent caching
• Staff task management and tracking
• Complete order lifecycle management
• Performance metrics and analytics

🏗️ Services Architecture

Service Purpose Global Variable Usage
DatabaseService Core data operations (CRUD) db await db.getOrders(), db.createStaff(), etc.
AuthService Authentication & authorization auth await auth.adminLogin(), auth.hasRole(), etc.
StaffService Staff management & tasks staff await staff.createStaff(), staff.acceptTask(), etc.
OrderService Order management & tracking orders await orders.createOrder(), orders.updateStage(), etc.

💼 Integration Examples

1. Admin Panel Login

async function handleAdminLogin(username, password) { try { const result = await auth.adminLogin(username, password); console.log('✅ Logged in as:', result.data.user.name); // Refresh UI based on role updateDashboard(); } catch (error) { console.error('❌ Login failed:', error.message); } }

2. Staff Panel Login

async function handleStaffLogin(staffId, pin) { try { const result = await auth.staffLogin(staffId, pin); console.log('✅ Staff logged in:', result.data.user.name); // Load staff tasks const tasks = await staff.getStaffTasks(staffId); displayTasks(tasks); } catch (error) { console.error('❌ Login failed:', error.message); } }

3. Create New Order

async function createNewOrder(formData) { try { const orderData = { customerId: formData.phone, customerName: formData.name, customerPhone: formData.phone, branch: auth.getBranch(), items: formData.items, totalAmount: formData.total, measurements: formData.measurements, deliveryDate: formData.deliveryDate, createdBy: auth.getUser().id }; const order = await orders.createOrder(orderData); console.log('✅ Order created:', order.orderId); showNotification('Order created successfully!'); } catch (error) { console.error('❌ Error:', error.message); } }

4. Assign Task to Staff

async function assignTaskToStaff(orderId, stageName, staffId) { try { const result = await orders.assignStaffToStage(orderId, stageName, staffId); // Notify staff about new task await notifyStaff(staffId, { type: 'task-assigned', orderId: orderId, stage: stageName }); console.log('✅ Task assigned to staff'); } catch (error) { console.error('❌ Assignment failed:', error.message); } }

5. Staff Complete Task

async function staffCompleteTask(staffId, taskId, completionData) { try { const result = await staff.completeTask(staffId, taskId, { notes: completionData.notes, images: completionData.images, qualityRating: completionData.rating }); console.log('✅ Task completed successfully'); // Task automatically moves to next stage // Performance metrics are updated automatically } catch (error) { console.error('❌ Error:', error.message); } }

6. Record Payment

async function recordPayment(orderId, amount, method) { try { const result = await orders.recordPayment(orderId, amount, method); const paymentStatus = await orders.getPaymentStatus(orderId); console.log('✅ Payment recorded'); console.log('Balance Due:', paymentStatus.balanceDue); } catch (error) { console.error('❌ Payment error:', error.message); } }

7. Real-time Listeners

// Listen to order updates const unsubscribe = orders.subscribe('update', (change) => { console.log('Order updated:', change.data); updateOrderInUI(change.data); }); // Listen to staff availability changes staff.subscribe('availability', (change) => { console.log('Staff availability:', change.data); refreshStaffList(); }); // Unsubscribe when done // unsubscribe();

🔐 Authentication & Authorization

Check User Permissions

// Check if user has specific role if (auth.hasRole('admin')) { console.log('User is admin'); } // Check if user has permission if (auth.hasPermission('manage_staff')) { console.log('Can manage staff'); } // Check if user can access branch if (auth.canAccessBranch('SAPTHALA Main')) { console.log('User can access this branch'); } // Get accessible branches const branches = auth.getAccessibleBranches(); console.log('Accessible branches:', branches); // Check authentication status if (auth.isAuthenticated()) { console.log('User is logged in'); }

Role-Based Access Control

Role Can Manage Can View Restrictions
Super Admin Everything (all branches) All data, all branches None
Admin Branch data, staff, orders Only owned branch Cannot manage other branches
Sub-Admin Limited staff & order ops Only owned branch Cannot add staff, manage payments
Staff Accept/complete tasks Own tasks only Cannot create orders

🔄 Real-Time Sync & Offline Support

Automatic Synchronization

// Database service automatically: // ✅ Caches GET requests // ✅ Queues offline operations // ✅ Syncs when online resumes // ✅ Retries failed operations up to 3 times // Check if online if (db.isOnline()) { console.log('Online - Real-time sync active'); } else { console.log('Offline - Using cached data'); } // Export cache for offline recovery const cacheData = db.exportCache(); localStorage.setItem('offline_cache', cacheData); // Import cache when resuming const cached = localStorage.getItem('offline_cache'); db.importCache(cached);

⚠️ Error Handling Best Practices

try { // Make API call const result = await db.createOrder(orderData); } catch (error) { if (error.message.includes('401')) { // Token expired - redirect to login window.location.href = '/login'; } else if (error.message.includes('403')) { // Permission denied showError('You do not have permission for this action'); } else if (error.message.includes('Network')) { // Offline showError('Connection lost. Data will sync when online.'); db.addToSyncQueue({ method: 'POST', endpoint: '/orders', body: orderData }); } else { // Other errors showError(error.message); } }

📦 Setup Instructions

Step 1: Include Service Files

<!-- In your HTML <head> or before </body> --> <script src="/database-service.js"></script> <script src="/auth-service.js"></script> <script src="/staff-service.js"></script> <script src="/order-service.js"></script>

Step 2: Wait for Initialization

// Wait for services to initialize window.addEventListener('load', async () => { // Give services 1 second to initialize await new Promise(resolve => setTimeout(resolve, 1000)); // Now you can use: auth, db, staff, orders console.log('✅ All services ready'); });

Step 3: Check Authentication

document.addEventListener('DOMContentLoaded', async () => { // Wait for auth service to initialize await new Promise(resolve => setTimeout(resolve, 500)); if (auth.isAuthenticated()) { console.log('✅ User is already logged in'); showMainApp(); } else { console.log('User needs to log in'); showLoginPage(); } });

Step 4: Listen to Auth Changes

// Subscribe to authentication events auth.subscribe((change) => { if (change.event === 'login') { console.log('User logged in:', change.data.user); showMainApp(); } else if (change.event === 'logout') { console.log('User logged out'); showLoginPage(); } else if (change.event === 'user-update') { console.log('User data updated'); updateUserProfile(change.data.user); } });
